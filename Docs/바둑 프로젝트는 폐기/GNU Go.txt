GNU Go

이 프로그램은 바둑 프로그램인 GNU Go입니다. GNU Go의 개발 버전은 http://www.gnu.org/software/gnugo/devel.html에서 확인할 수 있습니다. 프로젝트에 도움을 주시고 싶다면 TODO 파일을 참고해 주세요.

[설치]
요약하자면, './configure; make' 명령어를 통해 GNU Go를 빌드할 수 있습니다. 선택 사항으로 (루트 권한으로) 'make install'을 실행하면 /usr/local/bin에 설치되며 매뉴얼 페이지(man page)도 함께 설치됩니다. 또한 CGoban을 설치하는 것을 권장합니다. 자세한 내용은 INSTALL 파일을 확인하세요.

[문서]
사용자 문서는 터미널에서 'gnugo --help' 또는 'man gnugo'를 실행하여 확인할 수 있습니다. Texinfo 문서에는 사용자를 위한 지침뿐만 아니라 프로그래머와 개발자를 위한 GNU Go의 알고리즘 및 함수 기록이 포함되어 있습니다. info 리더나 emacs를 사용하여 읽거나, doc/ 디렉토리에서 'make gnugo.dvi' 또는 'make gnugo.ps'를 실행하여 인쇄용 문서를 얻을 수 있습니다. 또한 'makeinfo --html gnugo.texi' 명령으로 HTML 문서를 만들 수도 있습니다.

[CGoban을 통한 실행]
CGoban은 GNU Go를 실행하는 매우 좋은 방법입니다. X 윈도우 시스템에서 아름다운 그래픽 사용자 인터페이스(GUI)를 제공합니다.
1. CGoban을 시작합니다.
2. 제어판에서 "Go Modem"을 선택하여 Go Modem Protocol 설정을 엽니다.
3. 플레이어 중 하나(또는 둘 다)를 "Program"으로 선택하고 GNU Go가 설치된 경로를 입력합니다.
4. 규칙(Rules Set)을 Japanese로 설정해야 치석(handicap)이 제대로 작동합니다.
5. 보드 크기와 치석을 설정하고 확인을 누르면 준비가 완료됩니다.

참고: GNU Go 경로 지정 시 '--quiet' 같은 명령줄 옵션을 함께 줄 수 있습니다. 덤(komi)의 경우 GMP 프로토콜은 전달 기능이 없으므로, 명령줄 옵션(--komi)으로 지정하지 않으면 GNU Go가 기본값(호선 5.5, 접바둑 0.5)으로 추측하게 됩니다.

[ASCII 인터페이스]
CGoban이 없어도 기본 ASCII 인터페이스로 플레이할 수 있습니다. 터미널에 'gnugo'를 입력하면 바둑판이 그려집니다. 'help'를 입력하여 옵션 목록을 볼 수 있습니다. 게임이 끝나고 두 번 패스하면 계가 과정을 안내합니다.

[Emacs 모드]
Emacs 내에서도 GNU Go를 실행할 수 있습니다(Emacs 20.4 이상 필요). 커서 화살표 키로 돌을 놓을 수 있는 장점이 있습니다. 'interface/gnugo.el' 파일을 로드한 후 'M-x gnugo'로 시작합니다.

[Go Modem Protocol (GMP)]
GMP는 표준 프로토콜로 개발되었으며, CGoban이 이를 지원하므로 개발자는 GUI 구현에 신경 쓰지 않고 엔진 개발에 집중할 수 있었습니다.

[주요 옵션]
* --help, -h: 도움말 및 파라미터 기본값(레벨, 캐시 크기 등) 출력.
* --level LEVEL: 실력과 속도 조절. 기본값은 10이며, 낮출수록 빨라지지만 정확도는 떨어짐.
* --quiet, --silent: 저작권 메시지 등 출력 억제.
* --mode MODE: 실행 모드 강제 설정 ('ascii', 'gmp', 'gtp').
* --japanese-rules / --chinese-rules: 일본식 또는 중국식 규칙 설정.

[저작권 및 라이선스]
대부분의 파일은 Free Software Foundation의 저작권 소유이며 GNU 일반 공중 사용 허가서(GPL)를 따릅니다. 단, gtp.c 및 gtp.h 파일은 GTP 프로토콜 확산을 위해 보다 완화된 라이선스로 제공되어 자유로운 사용이 가능합니다.
